# 轻量化服务器文件整理纪要

> **日期**：2026-04-24  
> **执行人**：WorkBuddy（Qclaw）  
> **服务器**：119.29.84.148（腾讯云轻量应用服务器，4核4G，Ubuntu 24.04）

---

## 一、背景

服务器上部署了多个项目（桃悦OS、研策部工作平台），经过多次迭代和迁移后，`/var/www/` 目录存在以下问题：

1. **根目录残留**：桃悦OS v2.8（Next.js版）的旧代码占 727M（node_modules、public/、server/ 等）
2. **项目散落**：weekly-api 后端在 `/home/ubuntu/weekly-api/`，与其他项目分离
3. **权限混乱**：部分目录 root 所有，部分 ubuntu 所有
4. **Nginx 配置重复**：HTTP 和 HTTPS 两个 server 块各写了一遍相同的 location（~120行）
5. **磁盘压力**：`/var/www/` 占用 3.6G，大量空间被无用文件占用

---

## 二、优化方案

### 方案选型

| 维度 | 方案A（子域名分组） | 方案B（项目平铺）← 采用 |
|------|---------------------|------------------------|
| 域名 | 新增 work.moodway.top | 保持现有 www.moodway.top |
| 目录层级 | 3层（研策部工作平台/周报系统/weekly） | 2层（weekly/） |
| SSL 证书 | 需申请新证书 | 现有证书覆盖 |
| DNS 配置 | 需新增 A 记录 | 无需改动 |
| 改动风险 | 中（DNS+SSL+Nginx） | 低（仅Nginx+文件） |

**决策**：采用方案B，理由：
- 不需要改 DNS 和 SSL，零外部依赖
- 英文路径避免脚本编码问题
- 改动范围最小，可逐步优化

### 目标结构

```
/var/www/
├── moodway/           # 首页 + KPI静态页
│   ├── index.html     # 工作平台入口
│   └── kpi/           # KPI统计页
├── weekly/            # 周报前端（纯HTML）
├── weekly-api/        # 周报后端（Express + SQLite）
│   ├── server.js
│   ├── ecosystem.config.cjs
│   └── data/weeklydata.db
├── taoyue-os/         # 桃悦OS（reader.moodway.top）
├── drawing-review/    # 图纸审核（前端+后端）
└── rsshub/            # RSSHub
```

## 三、执行步骤

### 步骤1：清理根目录残留 ✅

**删除内容**：

| 类型 | 路径/文件 | 大小 |
|------|----------|------|
| 目录 | `/var/www/node_modules/` | 727M |
| 目录 | `/var/www/public/` | 桃悦OS v2.8 前端 |
| 目录 | `/var/www/server/` | 桃悦OS v2.8 后端 |
| 目录 | `/var/www/scripts/` | 构建脚本 |
| 目录 | `/var/www/html/` | Nginx 默认页 |
| 文件 | package.json, package-lock.json | — |
| 文件 | ecosystem.config.js, CHANGELOG_v2.8.md, PROJECT.md | — |
| 文件 | .env.example, .gitignore | — |

### 步骤2：迁移 weekly-api ✅

**操作**：
1. `cp -a /home/ubuntu/weekly-api /var/www/weekly-api`（复制到新位置）
2. 修改 `ecosystem.config.cjs` 中的 `cwd` 和 `DB_PATH`
3. PM2 重启并验证数据正常（14个项目、28个文档）
4. 更新 cron 备份路径
5. 删除旧目录 `/home/ubuntu/weekly-api/`

**踩坑记录**：
- `pm2 start server.js --name xxx` 不会自动设置 cwd，exec cwd 仍然是当前用户 home 目录
- 必须通过 `ecosystem.config.cjs` 显式指定 `cwd: '/var/www/weekly-api'`
- API 返回格式是 `{"projects":[...]}` 不是 `{"data":"[...]"}`，python 解析脚本需匹配实际结构

### 步骤3：统一权限 ✅

```bash
sudo chown -R ubuntu:ubuntu /var/www/
```

所有目录统一为 `ubuntu:ubuntu`，避免后续操作碰权限问题。

### 步骤4：Nginx 配置去重 ✅

**优化前**（~120行）：
- HTTP server 块：8 个 location
- HTTPS server 块：同样的 8 个 location
- 修改任何路由需要改两处

**优化后**（~45行）：
- 公共 location 抽到 `/etc/nginx/snippets/moodway-locations.conf`
- 主配置 `/etc/nginx/conf.d/reader-os.conf` 通过 `include` 引用
- 删除了与 `/weekly` 完全重复的 `/app` 路由

**Nginx 配置文件清单**：

| 文件 | 用途 |
|------|------|
| `/etc/nginx/conf.d/reader-os.conf` | 主配置（HTTP重定向 + HTTPS + IP直连） |
| `/etc/nginx/snippets/moodway-locations.conf` | 公共路由（/api, /weekly, /kpi, /reader, /drawing） |

|------|------|
| `/var/www/weekly-api/data/weeklydata.db` | 周报系统主数据库（SQLite） |
| `/home/ubuntu/backups/weeklydata-*.db` | 每日自动备份 |

### Cron 任务

```bash
crontab -l
# 0 2 * * * cp /var/www/weekly-api/data/weeklydata.db /home/ubuntu/backups/weeklydata-$(date +\%Y\%m\%d).db
# 0 3 * * * find /home/ubuntu/backups/ -name "weeklydata-*.db" -mtime +30 -delete
```
